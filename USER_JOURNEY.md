# ArcLink User Journey

> AUDIT TARGET — generated by the Ralphie journey/gaps audit
> (`./ralphie.sh --arclink-user-journey-audit`). This file is intentionally a
> stub at launch so the audit produces the atlas from fresh source evidence
> rather than polishing a pre-written draft.

## Audit mandate

Produce the complete end-to-end ArcLink experience story: every user journey
and path, joint by joint, surface by surface, across all systems and services —
public entry, Raven, Telegram, Discord, billing, provisioning, Control Node,
Shared Host, ArcPods, dashboards, Hermes, knowledge/memory, workspace, sharing,
provider/refuel, admin/operator, recovery, and trust boundaries. Include happy
paths, choice points, alternate paths, error/retry paths, access boundaries,
and cross-system handoffs.

Write it so it feels simple and mind-blowing at the human layer while staying
honest: describe the ideal product contract, but never present unproven
live/external behavior as already verified — mark those as proof-gated and
record them in `GAPS.md`.

Reference (input only, not ground truth):
`research/seed-user-journey-draft.md`. Improve on it, contradict it, or discard
it as the source evidence dictates.
