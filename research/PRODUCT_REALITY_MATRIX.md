# Product Reality Matrix

Status values:

- `real`: Static public-repo evidence supports the claim.
- `partial`: Some implementation exists, but the full journey, naming, tests,
  or UI truth is incomplete.
- `gap`: The stated contract is not implemented or is contradicted by code.
- `proof-gated`: The claim depends on live/external services or credentials not
  authorized for this PLAN pass.
- `policy-question`: The right product behavior is not decidable from code.

`research/RALPHIE_ARCLINK_PRODUCT_REALITY_AND_JOURNEY_STEERING.md`. It must be
updated as repairs land. Rows split the product claims so BUILD can move each
claim to `real`, `proof-gated`, or `policy-question` without hiding gaps inside
large grouped statements.
