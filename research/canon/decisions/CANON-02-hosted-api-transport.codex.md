- RECOMMENDATION: Change `python/arclink_api_auth.py:_session_hash_pepper()` so the fixed dev pepper is allowed only when `ARCLINK_SESSION_HASH_PEPPER` is absent, `ARCLINK_SESSION_HASH_PEPPER_REQUIRED` is false, and `ARCLINK_BASE_DOMAIN` is explicitly local/test (`localhost`, `127.0.0.1`, `::1`, `example.test`, or `*.test`). Blank/unset domain should raise `ArcLinkApiAuthError`, same as production. Update `tests/test_arclink_api_auth.py` for blank/unset fail-closed and explicit-local fallback; update `docs/API_REFERENCE.md` and env examples to say dev fallback requires an explicit local/test domain.
- SYMPHONY ANCHOR: `Secrets, Keys, And Rotation` + `Configuration, Schema, And Migration`: “Every credential should have status without disclosure” and generated config should detect “stale, missing, deprecated, or incompatible values before services start.”
- RATIONALE: Current code raises only when the required flag is true or the domain looks production-like, so a blank domain falls through to `arclink-dev-session-hash-pepper` (`python/arclink_api_auth.py:275-287`). Canonical deploy lanes already generate/pass a pepper and set `ARCLINK_SESSION_HASH_PEPPER_REQUIRED=1` (`compose.yaml:77-78`, `bin/docker-entrypoint.sh:520-521`). Treating blank as invalid makes the auth secret source-owned and fail-closed while preserving explicit localhost/test development.

Making `ARCLINK_SESSION_HASH_PEPPER_REQUIRED` default true everywhere is stricter but unnecessarily breaks explicit local/test direct-use. The recommended split keeps dev simple only when dev is declared.
- EFFORT / BLAST-RADIUS: low; touches `python/arclink_api_auth.py`, focused auth tests, API docs, and env examples.

### DECISION 2: Trusted proxy CIDRs must be separate from admin/client CIDRs, and missing `X-Forwarded-For` must fail closed
- RECOMMENDATION: Add `ARCLINK_TRUSTED_PROXY_CIDRS` as the only source for peers whose forwarded headers are trusted. Redefine `ARCLINK_BACKEND_ALLOWED_CIDRS` as the resolved admin/control client allowlist. In `python/arclink_hosted_api.py`, replace `_remote_ip_from_headers()` with a resolver that: if `remote_addr` is in `ARCLINK_TRUSTED_PROXY_CIDRS`, requires `X-Forwarded-For` and returns its first IP; if missing/malformed, returns no client and the CIDR gate 403s; if peer is not a trusted proxy, ignores forwarded headers and evaluates `remote_addr` directly. Update Compose/deploy config generation to write both values explicitly, and replace/prove the Next API rewrite (`web/next.config.ts:7-12`) with a source-owned proxy path that forwards `X-Forwarded-For`. Add local regressions for trusted-proxy-without-XFF denial, trusted-proxy-with-XFF allow/deny, and untrusted spoof rejection. Gate real ingress behavior under `PG-INGRESS`.
- SYMPHONY ANCHOR: `API, Webhook, And Extension Contracts`: “Whenever a new surface is added, it should declare whether it is Captain-facing, Operator-facing, Agent-facing, worker-facing, or internal-only. That declaration should decide auth, audit, redaction, rate limiting, docs, and tests.”
- RATIONALE: Current `ARCLINK_BACKEND_ALLOWED_CIDRS` is used both to trust the direct peer for forwarded headers and to authorize the final client (`python/arclink_hosted_api.py:633-660`, `4090-4100`). That means a trusted proxy with no `X-Forwarded-For` collapses authorization to the proxy IP. Splitting proxy trust from admin policy matches “operator owns hosts/policy,” gives one system truth, and fails closed with redacted 403 evidence instead of silently widening admin access.
- TRADEOFFS & ALTERNATIVES: Keeping one CIDR variable is backward-compatible but preserves the bypass. Trusting Next/edge behavior by assumption is not source-owned proof. Requiring XFF from every loopback peer would break direct local/admin calls. The residual risk is operator misclassification of proxy CIDRs; config docs, deploy rendering, health diagnostics, and `PG-INGRESS` proof should make that visible.
- EFFORT / BLAST-RADIUS: med; touches hosted API config/resolution, deploy/Compose env generation, API docs, web proxy behavior, and hosted API/web transport tests.
