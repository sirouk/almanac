## CANON-08 — Codex (GPT-5.5 xhigh) decision recommendations (symphony-anchored)

### DECISION 1: `operator_actions.request_source == "operator-raven"` is only a string convention
- RECOMMENDATION: Demote `request_source` to audit/display only and add a real authorization envelope for high-authority `operator_actions`. Current code stores caller-supplied `request_source` verbatim (`python/arclink_control.py:8409-8464`), Operator Raven writes `"operator-raven"` (`python/arclink_operator_raven.py:1308-1314`, `:1582-1588`), and the provisioner accepts that string as the gate (`python/arclink_enrollment_provisioner.py:2308-2313`). Add schema columns such as `actor_id`, `authorization_kind`, `authorization_payload_json`, `authorization_mac`, `authorized_at`, and `authorization_expires_at`, or an equivalent linked authorization table. Mint the authorization only from the shared server-side action broker after operator allowlist/session, nonce or approval-code confirmation, action allowlist, target binding, and reason capture. Bind the MAC to `action_kind`, `requested_target` or target hash, `actor_id`, source surface, confirmation id/nonce id, and expiry using a private-state `ARCLINK_OPERATOR_ACTION_AUTH_SECRET`. Change the provisioner gate to verify that envelope before claiming `upgrade` or `pin-upgrade`; missing, expired, mismatched, or tampered auth marks the row `failed` with a redacted operator notice. Keep old rows preserved as evidence, but fail them closed and require requeue. Add local regressions proving forged `request_source="operator-raven"` fails, valid Raven auth succeeds, target tampering fails, and old-state fixtures fail clearly.
- SYMPHONY ANCHOR: `Identity, Access, And Session Governance` — “Admin and Operator actions should never trust client-asserted privilege… without server-side authorization and typed action resolution.” Also `Admin Dashboard, API, And CLI Control` — “Mutating actions should converge on one action model with actor, reason, scope, nonce/confirmation, redaction, result, and rollback/repair guidance.”
- RATIONALE: This makes `arclink_control.py` the local source owner for the queue contract instead of letting each producer define authority by convention. It preserves state by recording rejected rows rather than deleting them, fails closed before root maintenance runs, gives every surface the same action truth, and keeps host upgrades under operator-owned policy. Local proof is unit/fixture coverage; live execution remains behind `PG-UPGRADE` / `PG-HERMES`.
- TRADEOFFS & ALTERNATIVES: I rejected leaving the string allowlist plus “current writers are safe”; that will rot as dashboard, CLI, and future APIs add producers. I also rejected a simple enum or DB check constraint, because any writer could still choose the privileged enum. Residual risk remains for code that can read the signing secret and write the DB; that is a narrower trusted-host boundary and should be documented until the action broker is isolated further.
- EFFORT / BLAST-RADIUS: high — touches `arclink_control.py` schema/helpers, `arclink_operator_raven.py`, `arclink_enrollment_provisioner.py`, admin/API/CLI action producers as they converge, and regression fixtures.

### DECISION 2: `host_readiness.check_ingress_strategy` always returns ok
- RECOMMENDATION: Make it a hard local configuration preflight for the declared ingress mode, not a live external ingress proof. Today both Cloudflare and fallback branches return `ok=True` (`python/arclink_host_readiness.py:154-161`) and `ready` includes that check (`:179-183`). Change `check_ingress_strategy` to read `ARCLINK_INGRESS_MODE` and fail on invalid or incomplete selected mode. For `domain`, pass only when Cloudflare token/ref plus zone are present, or when an explicit operator-approved local/Traefik fallback flag is set and surfaced as `traefik_local` / `local_only`. For `tailscale`, pass only when the required Tailscale DNS/path shape is configured. Do not call Cloudflare or Tailscale from readiness; instead expose `requires_live_proof=PG-INGRESS` in the check detail or adjacent operator snapshot evidence. Add tests for domain-missing-Cloudflare fails, explicit local fallback passes with local-only detail, tailscale-missing-DNS fails, and live proof remains a separate evidence state.
- SYMPHONY ANCHOR: `Fleet, Provisioning, Ingress, And Recovery` — “Ingress is either domain/Cloudflare/Traefik with wildcard subdomains or Tailscale path routing, with clear teardown evidence.” `Notifications, Incidents, And Evidence` — “A clear split between local dry-run proof, authorized live proof, policy decision, and residual-risk acceptance.”
- RATIONALE: This preserves the no-mutation host-readiness role while stopping a production host from reporting ready when its chosen ingress policy is impossible. The operator owns ingress mode and fallback acceptance; local readiness proves local config, while `PG-INGRESS` proves external DNS/TLS/provider behavior. The dashboard, CLI, and Raven can then show the same truth: locally ready, live-proof pending, or blocked.
- TRADEOFFS & ALTERNATIVES: I rejected keeping the current always-ok marker because it hides misconfiguration. I also rejected making host readiness perform live Cloudflare/Tailscale mutations or reachability checks; that belongs to the named proof gate and evidence layer. Residual risk: passing local config still does not prove public reachability, so deployment copy must not call ingress complete until `PG-INGRESS` evidence exists.
- EFFORT / BLAST-RADIUS: medium — touches `python/arclink_host_readiness.py`, readiness tests, operator snapshot/dashboard copy, and possibly deploy/reconfigure config validation.

### DECISION 3: hosted API `source_ip` is spoofable audit metadata, not authority
- RECOMMENDATION: Resolve this in the CANON-02 proxy/header trust model, then wire CANON-08 to that single resolver. Current hosted API already has `_remote_ip_from_headers`, which trusts forwarded headers only from a trusted direct peer (`python/arclink_hosted_api.py:636-653`) and uses it for CIDR/rate-limit decisions (`:4090-4097`, `:4112-4119`), but fleet enrollment callback bypasses it and passes raw `x-real-ip` / `x-forwarded-for` into `consume_fleet_enrollment` (`:2089-2108`). Pass `remote_addr` into `_handle_fleet_enrollment_callback`, compute a trusted origin through the central resolver, and store only that resolved origin. Add metadata such as `source_ip_trust`, `direct_peer`, and maybe redacted `forwarded_chain` if useful. If no trusted peer context exists, record `unverified` or blank rather than client-supplied headers. Keep fleet enrollment authorization on the bearer enrollment token/HMAC; if any future path wants `source_ip` for authorization, require trusted origin and fail closed. Add regressions: untrusted direct spoofed `X-Forwarded-For` records the direct peer, trusted proxy records the forwarded client, missing remote address records unverified, and callback auth behavior stays token-based.
- SYMPHONY ANCHOR: `API, Webhook, And Extension Contracts` — “Versioned hosted API routes with OpenAPI/static-dynamic parity tests.” `Identity, Access, And Session Governance` — “Admin and Operator actions should never trust client-asserted privilege…” Applied here as: do not trust client-asserted network identity.
- RATIONALE: Only the hosted API/proxy layer knows whether a header is trustworthy; fleet enrollment should receive a resolved origin plus trust metadata, not parse headers itself. This keeps evidence redacted and useful without turning spoofable audit text into policy. It also aligns rate limits, CIDR gates, audit metadata, and operator surfaces to one system truth.
- TRADEOFFS & ALTERNATIVES: I rejected patching `consume_fleet_enrollment` to interpret headers, because it lacks proxy context. I also rejected making the callback CIDR-protected by default; remote workers need to call it through the public or private ingress path, and the enrollment token is the actual capability. Residual risk remains if a trusted reverse proxy forwards unsanitized client headers; close that with CANON-02 proxy config tests plus `PG-FLEET` / `PG-INGRESS` live proof.
- EFFORT / BLAST-RADIUS: low to medium — touches `python/arclink_hosted_api.py`, fleet enrollment audit/metadata expectations, hosted API tests, and reverse-proxy/live-proof documentation.
