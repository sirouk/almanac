## CANON-09 — Codex (GPT-5.5 xhigh) decision recommendations (symphony-anchored)

### DECISION 1: Torn-down-row/global DNS unique-index collision
- RECOMMENDATION: Keep DNS hostnames permanently reserved by deployment prefix. Do not make `idx_arclink_dns_records_host_type` partial and do not delete/recycle torn-down DNS rows by default. Add an explicit preflight in `persist_arclink_dns_records` before the insert at `python/arclink_ingress.py:74`: if `(LOWER(hostname), UPPER(record_type))` is already owned by a different `record_id` or deployment, raise a sanitized typed error before SQLite emits a raw `IntegrityError`. Add a regression that seeds a torn-down conflicting row and proves fail-closed, state unchanged.
- SYMPHONY ANCHOR: `Fleet, Provisioning, Ingress, And Recovery`: “Ingress is either domain/Cloudflare/Traefik with wildcard subdomains or Tailscale path routing, with clear teardown evidence.” Also: “Rollback preserves state by default…”
- RATIONALE: The code already makes prefix reservation global via `idx_arclink_deployments_prefix` and `reserve_arclink_deployment_prefix` (`python/arclink_control.py:2023`, `python/arclink_control.py:3667`). That fits the vision: Operators own host namespace policy; Captains own Pods, not reusable hostnames. Preserving torn-down DNS rows keeps the Control DB as the same-truth ledger and prevents silent URL reassignment.
- TRADEOFFS & ALTERNATIVES: Rejected partial uniqueness such as `WHERE status != 'torn_down'`, automatic row deletion, or automatic prefix reuse. Those preserve convenience but weaken teardown evidence and can surprise Captains after rollback/recovery. Residual risk is manual DB corruption; the fix should make it fail clearly and point to an operator repair path.
- EFFORT / BLAST-RADIUS: med; touches `python/arclink_ingress.py`, schema/contract tests, and provisioning/action failure copy. No live Cloudflare mutation required unless an operator later adds an explicit hostname-release workflow gated by `PG-INGRESS`.

### DECISION 2: Persisted `proxied` drift
- RECOMMENDATION: Add `proxied INTEGER NOT NULL DEFAULT 1` to `arclink_dns_records` in `python/arclink_control.py`, with an idempotent old-state migration. Update `persist_arclink_dns_records` to write it and to preserve `provisioned` status only when hostname, type, target, and `proxied` all match. Update `dns_repair` row reads to use stored `proxied` instead of hard-coded `True` at `python/arclink_action_worker.py:205`. In the live executor, verify Cloudflare’s returned/applied `proxied` value matches desired before returning success; otherwise leave DB rows `desired` and fail closed.
- SYMPHONY ANCHOR: `Configuration, Schema, And Migration`: “Database schema changes are migration-aware, idempotent, reversible where practical, and tested against old-state fixtures.” `Secrets, Keys, And Rotation`: integrations must show configured/live-proof-pending/blocked states.
- RATIONALE: `DnsRecord.proxied` is part of the desired ingress contract and live Cloudflare apply already uses it (`python/arclink_executor.py:2582`), but the DB currently drops it (`python/arclink_control.py:1214`, `python/arclink_ingress.py:76`). Persisting it makes the DB the local source owner and keeps Raven/dashboard/repair paths aligned with the installed release truth.
- TRADEOFFS & ALTERNATIVES: Rejected keeping `proxied=True` implicit, because that hides drift and future policy changes. Rejected storing it only in metadata, because it is queryable provider state, not incidental evidence. Old rows can default to `true`, matching current behavior; any intentionally unproxied live records need an operator repair/proof pass.
- EFFORT / BLAST-RADIUS: high; touches schema migration, `python/arclink_ingress.py`, `python/arclink_action_worker.py`, executor metadata/proof tests, and `PG-INGRESS` live-proof expectations.

### DECISION 3: Public unused `tailscale_dns_name` / `tailscale_host_strategy` args
- RECOMMENDATION: Keep the public signature for release compatibility, but make the arguments meaningful. In `desired_arclink_ingress_records`, when `ingress_mode == "tailscale"`, call `arclink_tailscale_hostnames(prefix, tailscale_dns_name or base_domain, strategy=tailscale_host_strategy)` for validation, then return `{}` because there are no Cloudflare DNS records to persist. Add tests proving invalid Tailscale host/strategy fails closed.
- SYMPHONY ANCHOR: `API, Webhook, And Extension Contracts`: “Compatibility tests that prove old clients fail clearly or continue safely.” Also `API...`: contracts should be stable enough for “web, bots, workers, plugins, and future integrations to evolve without hidden breakage.”
- RATIONALE: Removing the args would break callers; continuing to ignore them lets direct callers pass an invalid Tailscale shape and get a false-success empty DNS plan. Validation-only use keeps the current return contract while making mode selection honest and fail-closed.
- TRADEOFFS & ALTERNATIVES: Rejected removal or rename until a versioned API break. Rejected returning synthetic Tailscale DNS records because it would mix Cloudflare DNS persistence with Tailscale path routing and risk accidental provider mutation.
- EFFORT / BLAST-RADIUS: low; touches `python/arclink_ingress.py`, focused ingress tests, and possibly docs comments around the function contract.

### DECISION 4: Non-empty internal commits in DNS helpers
- RECOMMENDATION: Do not remove internal commits in-place. Instead, codify them as the default contract: add keyword-only `commit: bool = True` to DNS DB helpers, keep current live callers on the default, and allow `commit=False` only for explicit caller-owned local transactions. Refactor `_mark_dns_status` / `mark_arclink_dns_provisioned` so row updates and `append_arclink_event(..., commit=False)` commit once together, avoiding a status-without-event split.
- SYMPHONY ANCHOR: `Notifications, Incidents, And Evidence`: “Every important background path should have an owner-visible state, a retry or repair path, and evidence that can be shared without secrets.” Also the north-star line: teardown should “preserve state by default and leave redacted evidence.”
- RATIONALE: DNS apply/teardown crosses an external boundary. Committing desired rows before Cloudflare mutation and committing provision/teardown evidence after provider results is the reliable phase model: a crash leaves visible state, not lost intent. Making `commit` explicit removes hidden transaction semantics without forcing live callers to hold DB transactions across network calls.
- TRADEOFFS & ALTERNATIVES: Rejected removing commits wholesale because it silently pushes durability to every caller and can erase evidence on failure. Rejected leaving the implicit behavior undocumented because future callers will keep guessing. Residual risk is caller misuse of `commit=False`; tests and docstrings should mark it local-only.
- EFFORT / BLAST-RADIUS: med; touches `python/arclink_ingress.py`, existing callers in `python/arclink_action_worker.py` and `python/arclink_sovereign_worker.py` only if they opt in, plus transaction/evidence regressions.
