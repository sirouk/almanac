## CANON-11 — Codex (GPT-5.5 xhigh) decision recommendations (symphony-anchored)

### DECISION 1: Production Chutes/Stripe admin clients and refund/cancel semantics
- RECOMMENDATION: Keep live provider-admin mutation fail-closed until explicit clients, policy, and proof gates are wired. Add `ExecutorChutesKeyClient` and `LiveStripeActionClient`, but inject them from `executor_for_fleet_host` only when `ARCLINK_DB_PATH` opens, mutation is explicitly enabled, secret refs are present, and `PG-PROVIDER` / `PG-STRIPE` are green. Chutes key rotation should be an Operator maintenance/compatibility lane, not the default Pod inference path; Pods should keep router credentials. Stripe refund must require a resolvable charge/payment_intent/invoice or policy-approved amount, not just a customer; cancel should default to period-end unless immediate cancellation is explicitly confirmed. Webhooks remain the entitlement source of truth. Current grounding: provider calls already fail without injected clients and DB in [python/arclink_executor.py](/root/arclink/python/arclink_executor.py:1217) and [python/arclink_executor.py](/root/arclink/python/arclink_executor.py:1342); production factory injects DB but no clients in [python/arclink_executor.py](/root/arclink/python/arclink_executor.py:158); dashboard already labels these as `PG-PROVIDER` / `PG-STRIPE` in [python/arclink_dashboard.py](/root/arclink/python/arclink_dashboard.py:72).
- SYMPHONY ANCHOR: Third-Party Integration Boundaries: “Stripe owns payment collection and subscription events; ArcLink owns entitlement interpretation, idempotency, gating, audit, and recovery copy.” Also: “Chutes or other providers own model execution; ArcLink owns router keys, model allowlists, fallback policy, budget enforcement, sanitized usage rows, and incident visibility.”
- RATIONALE: This keeps provider/product authority with the Operator, preserves ArcLink’s local source of truth, and avoids turning the executor into an ambient secret-powered billing/provider bot. It also matches the current fail-closed shape: `ChutesLiveAdapter` already proof-gates live key mutation in [python/arclink_chutes_live.py](/root/arclink/python/arclink_chutes_live.py:189), while current Stripe live code only covers checkout/portal, not refund/cancel, in [python/arclink_adapters.py](/root/arclink/python/arclink_adapters.py:66).
- TRADEOFFS & ALTERNATIVES: Rejected: wiring `STRIPE_SECRET_KEY` or a Chutes credential directly into executor mutation by default. Rejected: refund-by-customer, because it is underspecified and can hit the wrong charge. Residual risk is provider API mismatch; contain it behind fake adapter tests plus named live proof.
- EFFORT / BLAST-RADIUS: high; touches executor factory, provider adapters, action worker refund/cancel resolution, admin readiness copy, proof/evidence rows, and tests.

### DECISION 2: Replay ledger contract for compose, lifecycle, and DNS
- RECOMMENDATION: Add a typed replay contract, not a naive generic key ledger. For compose apply, change stable keys like `arclink:sovereign:compose:{deployment_id}` in [python/arclink_sovereign_worker.py](/root/arclink/python/arclink_sovereign_worker.py:1248) to content/generation keys such as `arclink:sovereign:compose:{deployment_id}:{intent_digest}`; same key plus same digest replays, same key plus different digest fails closed, and legitimate updates mint a new generation. For DNS, keep the Cloudflare lock but ledger desired-state by `deployment_id + zone + dns_digest`. For lifecycle, dedupe by action intent id, with teardown intent including `remove_volumes` and confirmation metadata. Use `arclink_operation_idempotency` as the base table in [python/arclink_control.py](/root/arclink/python/arclink_control.py:1464), extending only if generation/supersession fields are needed.
- SYMPHONY ANCHOR: API, Webhook, And Extension Contracts: “Action-worker contracts that define actor, action, target, reason, status, audit, dry-run, confirmation, retry, timeout, and rollback fields.” Fleet, Provisioning, Ingress, And Recovery: “Rollback preserves state by default and only deletes volumes with explicit destructive metadata and confirmation.”
- RATIONALE: Current live compose, DNS, and lifecycle paths execute without durable replay reservation, while fake compose already proves digest-bound replay in [python/arclink_executor.py](/root/arclink/python/arclink_executor.py:1077). A content/generation contract preserves valid re-applies and upgrades while still preventing stale retries, concurrent drains, and crash-after-provider-call ambiguity.
- TRADEOFFS & ALTERNATIVES: Rejected: binding all compose updates to the existing deployment-only idempotency key, because it would break legitimate rendered-intent changes. Rejected: no ledger beyond DNS lock, because it leaves weak evidence and retry semantics. Residual risk is Docker’s non-transactional partial apply; handle with health/reconcile evidence, not fake atomicity.
- EFFORT / BLAST-RADIUS: high; touches executor, sovereign worker, action worker, control schema/migration fixtures, DNS marking, local replay tests, and `PG-PROVISION` / `PG-INGRESS` / `PG-FLEET` proof expectations.

### DECISION 3: SSH TOFU default for fleet executor/bootstrap
- RECOMMENDATION: Split first-contact from steady-state. Change executor and steady-state fleet SSH defaults from `StrictHostKeyChecking=accept-new` to `StrictHostKeyChecking=yes`, requiring `ARCLINK_FLEET_SSH_KNOWN_HOSTS_FILE` for `ssh` executor mode. Keep TOFU only inside `deploy.sh control register-worker --bootstrap-remote`, where ArcLink should scan/display the host-key fingerprint, require operator confirmation or `--host-key-sha256`, then write the pinned key under `arclink-priv/secrets/ssh/known_hosts` and record redacted evidence. Host-key changes should fail closed and require an explicit re-attest/rotate-host-key flow. Current TOFU defaults are in [python/arclink_executor.py](/root/arclink/python/arclink_executor.py:138), [python/arclink_executor.py](/root/arclink/python/arclink_executor.py:550), and deploy bootstrap SSH paths like [bin/deploy.sh](/root/arclink/bin/deploy.sh:10029).
- SYMPHONY ANCHOR: “Operators own the universe: hosts, secrets, fleet, policy, upgrades, backups, live proof, emergency repair, and product rollout.” Secrets, Keys, And Rotation: “Stripe, Telegram, Discord, Chutes/provider, Cloudflare, Tailscale, SSH... credentials are stored only in private state or provider-owned stores.”
- RATIONALE: Operators own fleet admission, so first contact can be a deliberate enrollment ceremony, but ongoing executor, probe, image-sync, and inventory SSH should fail closed on unknown or changed host keys. This keeps fleet state auditable and prevents a first-connection MITM from becoming permanent operational truth.
- TRADEOFFS & ALTERNATIVES: Rejected: leaving `accept-new` everywhere for convenience. Rejected: requiring pre-pinned keys with no bootstrap exception, because it makes fresh worker admission brittle. Residual risk remains if the operator confirms an attacker’s fingerprint; reduce it with explicit fingerprint display, private evidence, and re-attestation docs.
- EFFORT / BLAST-RADIUS: med; touches executor SSH options, deploy register-worker/bootstrap, fleet inventory/probe/image-sync helpers, fleet-share SSH command rendering, docs, and regression tests.
