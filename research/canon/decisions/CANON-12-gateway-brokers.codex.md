Replace the current regex-plus-blocklist pass-through in [python/arclink_agent_process_helper.py](/root/arclink/python/arclink_agent_process_helper.py:334) and [python/arclink_docker_agent_supervisor.py](/root/arclink/python/arclink_docker_agent_supervisor.py:291) with one shared, versioned env contract module used by both sides. Unknown keys fail before `setpriv`/`Popen`. Add local tests proving `FOO=bar` is rejected by default and accepted only when operator-configured.

A pure hardcoded allowlist is safest but too brittle for real operator extensions; a private operator allowlist preserves escape-hatch semantics without making the request itself policy.

Explicit scope keeps state, makes policy operator-owned, and fails closed on ambiguous legacy state.
- TRADEOFFS & ALTERNATIVES: Fully preserving legacy user-pair behavior avoids breakage but hides a broad grant behind missing metadata. ### DECISION 3: agent-user-helper `chown -R` validate-then-act gap
- RECOMMENDATION: Replace shell `chown -R` at [python/arclink_agent_user_helper.py](/root/arclink/python/arclink_agent_user_helper.py:446) with an in-process no-follow ownership walk. Open the canonical home root/home with `O_DIRECTORY|O_NOFOLLOW|O_CLOEXEC`, verify `lstat`/`fstat`, walk with descriptor-relative operations, use `os.chown(..., dir_fd=..., follow_symlinks=False)` or `fchown` for directories, never follow symlink targets, and fail with a rejection incident on any race/path escape. Keep state; do not delete or recreate user files.
- SYMPHONY ANCHOR: Abuse, Safety, And Platform Boundaries — “Terminal/process/root-helper restrictions that treat shell execution as a brokered capability, never as ambient trust.”
- RATIONALE: Recursive ownership is root authority. A pinned fd-based walk makes the helper own the exact filesystem object it validated, avoids shell/path races, preserves existing homes, and produces local proof for symlink and race-style regressions.
- TRADEOFFS & ALTERNATIVES: `chown -R -h` is lower effort but remains path-based. A validate-then-shell approach is familiar but keeps the gap. The fd walk is more code and needs careful tests, but it best matches the trusted-host boundary.
- EFFORT / BLAST-RADIUS: med; touches agent-user-helper, tests, trusted executable expectations, and possibly docs for root-helper controls.

### DECISION 4: `record_rejection_incident` silent no-op on unsafe/OSError paths

If the sink is unsafe or unwritable, mutating endpoints return 503 before privileged work, health reports the incident sink unavailable, and a sanitized stderr line is emitted as last-ditch evidence. - SYMPHONY ANCHOR: Notifications, Incidents, And Evidence — “ArcLink should never fail silently.”
- RATIONALE: These rejections are the evidence trail for root/Docker boundaries. Best-effort logging is acceptable for low-risk telemetry, but not for GAP-019 controls. Failing closed makes the operator fix local state instead of running blind.
- TRADEOFFS & ALTERNATIVES: Keeping silent no-op maximizes availability but violates the evidence contract. Raising without caller handling risks ugly request failures. A strict, typed result with health integration gives observability without secret leakage.
- EFFORT / BLAST-RADIUS: med; touches rejection_incidents, all broker/helper rejection wrappers, health tests, and operations docs.
