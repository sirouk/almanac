## CANON-18 — Codex (GPT-5.5 xhigh) decision recommendations (symphony-anchored)

### DECISION 1: Notion verification-token first-caller authenticity under public Funnel
- RECOMMENDATION: Add an operator-minted one-time nonce to the Notion handshake URL, and gate signed Notion events until operator confirmation. `webhook-arm-install` should generate a high-entropy nonce, store only its hash in `settings`, and return a temporary URL like `/notion/webhook?install_nonce=...`. `Handler.do_POST` in [python/arclink_notion_webhook.py](/root/arclink/python/arclink_notion_webhook.py:339) should parse query params, pass the nonce into `handle_verification_token_post`, and reject missing/wrong nonce while preserving the armed window. After storing the token, clear the nonce/armed state as today. For non-handshake events, keep HMAC verification first, then require `notion_webhook_verified_at` before `store_notion_event`; otherwise return `412` and queue nothing. Update `deploy.sh notion-ssot` to arm before showing the exact URL, and add local tests for nonce-required install, one-shot nonce consumption, event rejection before `webhook-confirm-verified`, and accepted signed events after confirmation.
- SYMPHONY ANCHOR: **Identity, Access, And Session Governance**: “Rate limits, replay protection, nonce/confirmation, channel allowlists, and reason capture should be consistent...” Also **API, Webhook, And Extension Contracts**: “Idempotent webhook handlers for Stripe, bots, Notion...”
- RATIONALE: Current code has the race fixed with `BEGIN IMMEDIATE` in [handle_verification_token_post](/root/arclink/python/arclink_notion_webhook.py:211), but public Funnel still makes “first caller wins” a workflow trust problem because Funnel proxies to loopback and `backend_client_allowed` accepts loopback. A nonce makes the operator’s setup URL the local source of authority; confirmation-gating means a stored token is not enough to mutate Notion index state. Existing token state is preserved: installed-but-unconfirmed becomes blocked/warned, not deleted. Evidence remains redacted via `note_refresh_job`; live external proof belongs to `PG-NOTION`.
- TRADEOFFS & ALTERNATIVES: Only gating signed events would stop forged reindex events but still allow public DoS by consuming the first install window. Only adding a nonce would protect install but still let accidental/unconfirmed setup process events. Origin/IP checks are rejected because Funnel collapses public callers to loopback. Residual risk is operator leakage of the nonce-bearing URL; bounded by TTL and one-shot consumption.
- EFFORT / BLAST-RADIUS: med; touches [python/arclink_notion_webhook.py](/root/arclink/python/arclink_notion_webhook.py:211), [python/arclink_ctl.py](/root/arclink/python/arclink_ctl.py:2708), [bin/deploy.sh](/root/arclink/bin/deploy.sh:1003), health/status copy, and Notion webhook tests.

### DECISION 2: Webhook `/health` stays pre-auth
- RECOMMENDATION: Keep `/health` pre-auth as a minimal local/container liveness contract, but stop treating it as a public Funnel route. Leave the order asserted by [tests/test_loopback_service_hardening.py](/root/arclink/tests/test_loopback_service_hardening.py:73): `/health` before `_require_loopback_transport`. Keep the response static and non-sensitive: `{"ok": true, "service": "arclink-notion-webhook"}` only. Change the Funnel installer/health proof to publish and verify only `TAILSCALE_NOTION_WEBHOOK_FUNNEL_PATH` (`/notion/webhook`) to `http://127.0.0.1:$PORT/notion/webhook`, not root `/`; fail strict health if root `/` proxies to the webhook backend. Keep local health checks in [bin/health.sh](/root/arclink/bin/health.sh:2227) and Docker health checks unchanged.
- SYMPHONY ANCHOR: **Observability, SLOs, Capacity, And Scale**: “Health is layered...” and **Abuse, Safety, And Platform Boundaries**: “ArcLink should assume that public entry points will be poked, spammed, and misused.”
- RATIONALE: The current `/health` contract supports boring local monitoring, and changing it would break service/Docker health behavior. The unsafe part is exposure, not the local route. Path-limiting Funnel aligns public ingress with the actual webhook contract while preserving the same local source of truth for health. It fails closed: custom/root public exposure becomes a health warning/failure instead of silently publishing liveness and adjacent paths.
- TRADEOFFS & ALTERNATIVES: Authenticating `/health` was rejected because it breaks existing loopback/container health probes. Leaving root Funnel exposure was rejected because it makes `/health` public and contradicts the “only the configured Notion webhook route” health copy. Residual risk remains for manually configured non-Tailscale ingress; generated configs and health checks should flag root exposure, with external ingress proof under `PG-INGRESS` and Notion proof under `PG-NOTION`.
- EFFORT / BLAST-RADIUS: med; touches [bin/tailscale-notion-webhook-funnel.sh](/root/arclink/bin/tailscale-notion-webhook-funnel.sh:290), [bin/health.sh](/root/arclink/bin/health.sh:521), deploy Funnel detection, and health/deploy regression tests. The webhook service API remains stable.
