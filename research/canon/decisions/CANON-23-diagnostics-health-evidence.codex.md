## CANON-23 — Codex (GPT-5.5 xhigh) decision recommendations (symphony-anchored)

### DECISION 1: `run_diagnostics(live=...)` must not silently imply live provider connectivity
- RECOMMENDATION: Keep `python/arclink_diagnostics.py` as local, secret-safe presence/readiness diagnostics only. Do not add Stripe, Cloudflare/Tailscale, Chutes, Telegram, or Discord network calls inside `run_diagnostics`; current code only checks credential names and Docker/qmd state, and never branches on `live` ([python/arclink_diagnostics.py](/root/arclink/python/arclink_diagnostics.py:200)). Tighten the contract by changing the docstring and CLI help, and make `live=True` fail closed instead of no-op: keep the arg for compatibility, but append a failed `DiagnosticCheck(provider="live-proof", name="provider_connectivity", ok=False, detail="live provider validation is not run by diagnostics; use named PG-* live proof", live=True)` so `python3 -m arclink_diagnostics --live` exits nonzero. Add regression coverage that `live=True` fails closed and still redacts all secret values. Implement real provider connectivity only as named `bin/arclink-live-proof --journey external` runners under explicit `ARCLINK_PROOF_*` gates already modeled in [python/arclink_live_journey.py](/root/arclink/python/arclink_live_journey.py:217), with redacted evidence persisted through the live runner, not as dashboard snapshot side effects.
- SYMPHONY ANCHOR: **Third-Party Integration Boundaries**: “Every integration must have three visible states: configured and locally valid, configured but live-proof pending, or missing and blocked with the next operator action.” Also **Notifications, Incidents, And Evidence**: “A clear split between local dry-run proof, authorized live proof, policy decision, and residual-risk acceptance.”
- RATIONALE: This preserves the local source-owner split. Diagnostics owns local configured/missing state; live proof owns external provider truth. It also prevents a read-only dashboard/operator snapshot from becoming a networked provider probe, while making `--live` fail closed instead of creating false confidence. The existing live-proof path already has the correct shape: it composes diagnostics, named journey rows, evidence, and failure states, and external rows currently fail as `blocked_no_registered_runner` when no authorized runner exists ([python/arclink_live_runner.py](/root/arclink/python/arclink_live_runner.py:698)). That keeps operator-owned policy and external proof gates out of Captain/Pod runtime semantics.
- TRADEOFFS & ALTERNATIVES: Rejected wiring live provider calls directly into `run_diagnostics` because it would conflate local health with provider availability, risk rate limits or mutation-adjacent behavior, and make every surface that reads diagnostics depend on external services. Rejected leaving `live=True` as a no-op because it contradicts fail-closed evidence semantics. Abruptly removing the parameter is cleaner but more disruptive; a compatibility fail-closed check is safer. Residual risk remains that present credentials may be invalid until the relevant `PG-*` runner exists and passes; surfaces must label that as live-proof pending.
- EFFORT / BLAST-RADIUS: med. Low for contract cleanup in diagnostics, docs, and tests; medium when adding provider-specific proof runners. Touches `python/arclink_diagnostics.py`, `tests/test_arclink_diagnostics.py`, local-validation/live-proof docs, and later `python/arclink_live_runner.py`, evidence/dashboard/Raven surfaces for named `PG-STRIPE`, `PG-BOTS`, `PG-INGRESS`, and `PG-PROVIDER` proof status.
