## CANON-24 — Codex (GPT-5.5 xhigh) decision recommendations (symphony-anchored)

### DECISION 1: Unify `bootstrap.handshake` payload semantics across remote `init.sh` and host-side `bin/init.sh`
- RECOMMENDATION: Yes, unify the contract in CANON-18, but do not make `source_ip` a required client-produced field. Treat `bootstrap.handshake` as one versioned MCP schema with `unix_user`, optional `requester_identity`, explicit `auto_provision` defaulting false, optional `model_preset/channels`, and optional `source_ip` only as a trusted loopback override. Code-level: add `"auto_provision": false` to the host-side `bin/init.sh` payload at `bin/init.sh:450-454`; leave remote `init.sh` as `auto_provision: true` at `init.sh:265-269`; do not synthesize `source_ip` in remote `init.sh`. Keep `python/arclink_mcp_server.py:224-233` as the schema owner and keep `_request_source_ip()` / `_ensure_bootstrap_source_allowed()` at `python/arclink_mcp_server.py:1847-1861` as the origin authority. Add regressions proving: remote auto-provision works without `source_ip` and returns no `raw_token`; host-side manual handshake is explicitly `auto_provision=false` and still returns a pending token; spoofed `source_ip` is ignored unless `ARCLINK_ALLOW_LOOPBACK_SOURCE_IP_OVERRIDE=1`. Local proof: `bash -n init.sh bin/init.sh`, `python3 tests/test_loopback_service_hardening.py`, `python3 tests/test_arclink_auto_provision.py`, focused deploy-regression static checks. Named live proof for public/remote ingress stays `PG-PROVISION` plus `PG-INGRESS` when the public MCP path is exposed.
- SYMPHONY ANCHOR: `API, Webhook, And Extension Contracts`: “Versioned MCP schemas”. Also `Whole-System Traversal`: “Every step should have a local source owner, a local regression or dry-run proof where possible... and how it fails closed.”
- RATIONALE: This preserves the intended split: Captains can request remote auto-provision without host access, but Operators still own the host because `auto_provision=True` only queues operator approval and returns no raw token (`python/arclink_control.py:11287-11375`). The MCP server remains the local source owner for origin truth: it derives `source_ip` from transport and only honors a declared override from loopback when explicitly enabled (`python/arclink_mcp_server.py:1847-1856`). That fails closed, avoids trusting public client metadata, and keeps both producers aligned with one schema instead of two tribal payload shapes.
- TRADEOFFS & ALTERNATIVES: Requiring both producers to send both keys was rejected because remote `init.sh` cannot know a trustworthy source IP; forcing it to invent one would weaken audit and status gating. Removing `source_ip` from host-side entirely was also rejected because it remains useful for trusted local/SSH provenance when the operator deliberately enables loopback override. Residual risk is schema drift unless CANON-18 owns the documented schema and CANON-24 producer tests pin the two caller intents.
- EFFORT / BLAST-RADIUS: low. Touches `init.sh`, `bin/init.sh`, MCP schema/tests, auto-provision smoke/static tests; no DB migration and no state rewrite.
