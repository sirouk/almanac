## L1L2 - Codex implementation report

### L1 (single-platform env-starve)

Files/functions: `python/arclink_public_agent_bridge.py`
(`_load_gateway_config_for_platform`, `_unused_platform_env_prefixes`,
`_run_telegram`, `_run_discord`), plus flag propagation in `compose.yaml` and
`python/arclink_provisioning.py`.

Flag: `ARCLINK_BRIDGE_SINGLE_PLATFORM_CONFIG`, default `0`.

Behavior: the bridge still calls Hermes' real `load_gateway_config()` and never
constructs a replacement `PlatformConfig`. When enabled, it snapshots the
current process environment, removes only the unused platform prefix
(`DISCORD_*` for Telegram turns, `TELEGRAM_*` for Discord turns), calls the
Hermes loader, and restores the snapshot in `finally`. If the starved loader
raises, it restores the environment and reruns the same unstarved loader.
Worst case is no speedup.

### L2 (Telegram getMe cache)

Files/functions: `python/arclink_public_agent_bridge.py`
(`_bridge_getme_cache_secret`, `_bridge_getme_cache_key`,
`_bridge_getme_secure_cache_dir`, `_read_getme_cache`,
`_apply_cached_telegram_getme`, `_write_getme_cache`,
`_initialize_telegram_bot`), plus flag propagation in `compose.yaml` and
`python/arclink_provisioning.py`.

Flags: `ARCLINK_BRIDGE_GETME_CACHE`, default `0`;
`ARCLINK_BRIDGE_GETME_CACHE_TTL_SECONDS`, default `180`, capped at `300`;
`ARCLINK_BRIDGE_GETME_CACHE_DIR`, default
`/var/cache/arclink-public-agent-bridge/getme`.

Safety rails: cache keys are `HMAC(server_secret, bot_token)` hex, never raw
tokens or plain token hashes. The cache is disabled unless the directory is
outside Agent-writable roots, non-symlink, root-owned, exactly `0700`, and
writable by the root preload wrapper. The wrapper reads or refreshes only the
Telegram bot profile, then launches the real bridge as the normal runtime uid
with that non-secret profile preloaded. The server secret is read from existing
ArcLink private state/env only; no usable secret disables the cache. Missing,
stale, corrupt, insecure, unwritable, or exceptional cache paths all fail open
to live `bot.initialize()`.

D5 backstop: a cached Telegram `getMe` can only skip the identity probe. The
real send still must return a platform message id before notification delivery
marks the outbox row delivered, so revoked or rotated tokens fail closed at
send evidence.

### Tests added + results

Added focused local regressions to `tests/test_arclink_notification_delivery.py`
for:

- flags off preserving current config loading and live getMe;
- L1 env-starving unused platform variables and restoring them;
- L1 restoration plus unstarved fallback when the starved loader raises;
- L2 cache hit skipping live getMe with an HMAC key;
- L2 miss, stale, and corrupt entries failing open and refreshing;
- L2 insecure directory and missing-secret paths disabling the cache.

### Honest limits

The local suite stubs Hermes `gateway.*` and the Telegram library, so it proves
contract behavior, not runtime latency. Both speed flags are default off. Real
speedup, cache writability in the running container, and Telegram/Discord
delivery evidence must be confirmed through `PG-PUBLIC-AGENT-DELIVERY` or a
named live performance gate before claiming production latency gains.
