## W7 (CANON-08 D1) — Codex implementation report
### Changed
- `python/arclink_control.py`: `operator_actions` schema now has auth envelope columns. Added the single source owner for mint/verify: `operator_action_auth_secret*`, `_mint_operator_action_authorization`, and `operator_action_authorization_valid`. `request_operator_action` now accepts `authorization=...`, requires it for `upgrade` and `pin-upgrade`, and stores a server-minted HMAC envelope.
- `python/arclink_operator_raven.py`: after existing actor + confirmation gates, Raven now passes an explicit `operator-raven-confirmed` authorization payload for host upgrade, pin-upgrade, and upgrade-sweep queueing.
- `python/arclink_enrollment_provisioner.py`: replaced the `request_source == operator-raven` gate with `operator_action_authorization_valid`; invalid/missing/expired/tampered envelopes reuse the fail-closed action failure + operator notification path.
- Added report: `research/canon/phase1/W7.fix.md`.

### FINAL-PLAN mapping
- Schema columns: implemented idempotent ADD COLUMNs and create-table columns in `arclink_control.py`.
- Mint in chokepoint: `request_operator_action` signs action kind, target hash, actor id, auth kind, confirmation id/payload hash, and expiry with `ARCLINK_OPERATOR_ACTION_AUTH_SECRET` or a private-state key.
- Producer proof carried forward: Operator Raven passes actor id plus confirmation/idempotency evidence only after its actor and confirm gates pass.
- Consumer verification: provisioner recomputes and constant-time-compares the MAC before host/pin execution; `request_source` is now audit/display only.
- Fail closed/preserve state: unsigned, malformed, expired, and tampered rows are marked `failed`, notify the operator, and remain in `operator_actions`.

### Old hole closed (proof)
- A row with `request_source='operator-raven'` and no auth envelope no longer runs.
- Direct `request_operator_action(... action_kind='upgrade', request_source='operator-raven')` without `authorization` raises `PermissionError`.
- Tests mutate `authorization_mac`, `requested_target`, and `actor_id`; each fails with MAC mismatch and no execution.
- Pre-migration authless `operator_actions` rows are preserved by schema migration but fail closed.
