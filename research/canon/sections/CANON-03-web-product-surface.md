# CANON-03 â€” Web App & Product Surface

## PIECE
CANON-03 owns the **production customer-facing web app** (Next.js 15 App Router, `web/src/**`) and a **stdlib-only WSGI prototype product surface** plus the **shared product/env/copy contract** in Python. Tracked files owned:
- Next.js app: `web/src/app/{layout.tsx,page.tsx,manifest.ts}`, route pages `web/src/app/{login,onboarding,dashboard,admin,checkout/success,checkout/cancel,policy,privacy,terms,refunds}/page.tsx`, `web/src/app/globals.css`, components `web/src/components/{ui.tsx,pwa-register.tsx,marketing/{nav,footer,legal-page,marketing-home}.tsx}`, and the API client `web/src/lib/api.ts`.
- Build/config: `web/package.json`, `web/next.config.ts`, `web/playwright.config.ts`.
- Python: `python/arclink_product.py` (brand name, env resolution `resolve_env`, launch-phrase vocabulary), `python/arclink_product_surface.py` (the local WSGI prototype `handle_arclink_product_surface_request` + `ArcLinkSurfaceResponse`), `python/arclink_surface_contract.py` (the cross-surface copy/secret linter `surface_contract_issues`/`assert_surface_contract`).

The web app is a thin React client: every backend interaction flows through `web/src/lib/api.ts`, which hits the CANON-02 hosted API under prefix `/api/v1` (rewritten server-side by `next.config.ts` to an internal WSGI host). The Python product-surface module is a **self-contained loopback prototype with its own inline HTML** (NOT the Next.js app and NOT the hosted API); `arclink_product.py` is the only Python file here consumed by production paths (its `base_domain`/`chutes_default_model`/`launch_phrase` helpers are imported across the control plane).
