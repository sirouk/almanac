# CANON-16 — LLM Router & Providers

## PIECE
This piece is ArcLink's inference data plane and provider registry. It owns five tracked files (all present): `python/arclink_llm_router.py` (1917 lines) — a FastAPI ASGI app (`app = create_app()` at line 1917) exposing `GET /health`, `GET /v1/models`, `POST /v1/chat/completions`, run as `uvicorn arclink_llm_router:app` on port 8090; `python/arclink_model_providers.py` (122 lines) — a YAML+default-merged provider preset registry (chutes/opus/codex) with target/default-model/recommended-model/legacy-alias resolution; `python/arclink_chutes.py` (1042 lines) — model-catalog parsing, the per-deployment Chutes credential+budget boundary (`evaluate_chutes_deployment_boundary`), sanitized usage ingestion (`record_chutes_usage_event`), official-registration model, and fake key/inference helpers; `python/arclink_chutes_oauth.py` (361 lines) — PKCE-S256 OAuth connect/callback/disconnect helpers, in-memory stores, fake exchanger; `python/arclink_chutes_live.py` (443 lines) — a secret-ref boundary adapter over Chutes account/usage/key/OAuth-introspect REST APIs with a fixture transport. The seam config `config/model-providers.yaml` is owned by CANON-27 but consumed here via `arclink_model_providers`. The router takes per-deployment `acpod_live_` Bearer keys, enforces allowlist/replacement/budget/rate/concurrency policy, reserves budget over the max-cost fallback candidate, relays (non-streaming + SSE) to a central Chutes upstream with a fallback cascade, and settles sanitized usage — never persisting prompts/completions.

## CODE vs COMMENT/DOC/NAME DRIFT
- **Prior doc `research/ground-truth/03-llm-router-providers.md:11,18` claims the router has "1755 lines" and a single "60s timeout".** CODE: the file is 1917 lines (`wc`), and timeouts are split into connect=5/read=300/write=30/pool=5 seconds with a per-loop pooled client + startup warmup (`arclink_llm_router.py:44-50,1225,1281`). The 60s figure is stale.
- **Prior doc :92 and :167 claim live proof is "gated by `ARCLINK_LLM_ROUTER_LIVE_CHUTES_PROOF=1` + `PG-PROVIDER`".** CODE: no such env var exists anywhere outside markdown (`rg` over `--type py` and non-md files returns nothing); `arclink_live_runner.py:580` uses `httpx.MockTransport`. The named gate variable is fictional; the substantive claim (no live call) holds, but via a mock transport, not an env gate.
- **Prior doc does NOT mention the `budget_policy=observe_only_unlimited` / `budget_status="unlimited"` Operator-Pod lane.** CODE: `arclink_chutes.py:16,711-715,767-772` introduces `OBSERVE_UNLIMITED_REMAINING_CENTS=10**12`; the router skips the reservation budget block when `budget_status=='unlimited'` (`arclink_llm_router.py:1149`). - **Default model name drift across the registry.** `arclink_model_providers.py:13` and `config/model-providers.yaml:5` use `moonshotai/Kimi-K2.6-TEE` and `opus → claude-opus-4-7`, `codex → gpt-5.5`. - **Compose vs code default budget.** `compose.yaml:584` sets `ARCLINK_LLM_ROUTER_DEFAULT_MONTHLY_BUDGET_CENTS:-2500`; the code default is `0` (`arclink_llm_router.py:250`). Effective deployed default is 2500. (Prior doc :86 already flagged this — confirmed still true.)

## ADVERSARIAL SELF-CHECK
1. **"The router never persists prompts/completions."** Falsified if any path stores raw body. Checked: `arclink_llm_usage_events` insert only writes token counts + redacted `error_summary` + metadata (`:1372`); `error_summary` from `upstream.text` is run through `redact_then_truncate` limit 300 (`:668,:1394`). Residual risk: a non-secret prompt fragment could survive redaction inside a 300-char error string — secrets regex catches keys, not arbitrary user text.
2. **"Reservation always covers the most expensive candidate."** Could be falsified if `_fallback_reservation_pricing` returned the primary on empty fallbacks incorrectly. Code picks `max(choices, key=reserved_cents)` (`:585`) and has an empty-list guard (`:573`). Confident.
3. **"OAuth/live adapters are unwired."** Falsified if any production module imports them. `rg -l` over `--type py` shows imports only from `tests/`. Confident, but a runtime dynamic import (importlib) elsewhere could evade `rg` — I did not grep for string-based dynamic imports.
4. **"The `unlimited` lane cannot be set by an untrusted user."** This depends on who writes `chutes.budget_policy=observe_only_unlimited` into `arclink_deployments.metadata_json`. I traced the router CONSUMER (`:1149`) and boundary PRODUCER (`arclink_chutes.py:711`) but did not fully trace who STAMPS that metadata on the operator deployment — that producer is in CANON-08/CANON-14 territory. If a Captain Pod's metadata could carry it, budget enforcement would silently disable.
5. **"Both ends of the model-providers YAML seam match."** I verified key names match config:2-43 vs loader; I did not enumerate every CANON-27 writer of that file, only the schema/keys consumed here.

## RISKS (severity-ranked, code-cited)
- **MEDIUM — Budget fail-open via `unlimited` policy.** `arclink_llm_router.py:1149` and `arclink_chutes.py:767-772` disable the reservation budget gate and report `remaining_cents=10**12` whenever deployment metadata carries `budget_policy=observe_only_unlimited`. If that flag is settable on a non-operator deployment, inference becomes uncapped while still "metered." The stamping producer was not traced in this piece.
- **MEDIUM — Reservation is advisory, not atomic.** Concurrency/budget checks (`_open_reserved_count` `:834`, `remaining_cents` `:1149`) and the INSERT are separate statements on a SQLite connection with `busy_timeout` but no explicit transaction/row-lock around the read-then-reserve, so two concurrent requests can both pass the concurrency/budget check before either reserves (TOCTOU). Bound by `deployment_concurrency_limit` default 4, so blast radius is small.
- **LOW — Write on the authenticated read path.** `verify_llm_router_key` UPDATEs `key_hash`+`last_seen_at` and commits on every successful auth (`arclink_control.py:6810`); under high QPS this is an extra write per request and a `busy_timeout` contention source.
- **LOW — Failed-request settlement charges `actual_cents=0` but still releases the reservation as `failed`.** `_record_router_usage` (`:1331,:1435`) never bills failures — correct for cost, but a deployment can burn the whole concurrency budget on failing upstreams with zero `used_cents` advance, so budget exhaustion never triggers from pure failures (intended, noted for completeness).
- **LOW — Catalog `mark_missing_unavailable` can silently flip an in-use model to `unavailable`.** `upsert_model_catalog` (`arclink_control.py:6603`) marks any active model not in the latest catalog response `unavailable`; combined with auto-promote (`:617`) a transient bad catalog fetch could reroute live traffic. Startup refresh failures are caught and stored as `failed` (`:1825`), so the catalog is left stale rather than emptied — partial mitigation.
- **INFO — `_upstream_status_is_retryable` treats ALL ≥500 as retryable** (`:665`) in addition to the configured set; a 501/505 from the upstream will burn fallback candidates. Intended generosity.
- **INFO — Fuel notices are best-effort and swallow all exceptions** (`arclink_llm_router.py:1068`); a broken notification path is silent by design but unobservable from this piece.

## VERDICT
This piece provably does its job as a local-real inference control plane. Load-bearing strengths verified in code: per-deployment `acpod_live_` HMAC keys with legacy-hash migration and a single-row, status-gated verify; a fail-closed Chutes credential/budget boundary that rejects operator-shared keys, plaintext, and unscoped secret refs; max-candidate budget reservation with final-model settlement; key/deployment/user rate limits + concurrency cap; a complete non-streaming + pre-stream-fallback cascade with explicit no-replay-after-stream labeling; and sanitized usage/audit rows that never persist prompts or completions. The model-providers registry cleanly serves CANON-04 and CANON-19 with deterministic preset→target/model resolution and legacy-alias collapsing. Real weaknesses: the actual Chutes relay is never exercised live (the only "proof" harness uses `httpx.MockTransport`, and the prior doc's named live-proof env gate does not exist — GAP-031 is genuinely open); the `observe_only_unlimited` budget fail-open is new, real, and its trust source was not traced here; reservation is advisory (TOCTOU) rather than transactional; and the OAuth + live-account adapters, while well-shaped and secret-safe, are import-only by tests and proof-gated for every mutation. Net: the policy/accounting/fallback machinery is solid and well-cited; the live provider edge and the unlimited-Pod trust boundary are the remaining unproven seams.
