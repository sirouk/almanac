# CANON-17 — Academy / Crew / SOUL

## PIECE
This piece is the ArcLink **Academy** (specialist-formation subsystem), the **Crew Recipe** SOUL-overlay generator, and the **SOUL templates** that project ArcLink identity into an Agent's `SOUL.md`. It owns exactly seven tracked files:
- `python/arclink_academy_programs.py` (3121 lines) — control-plane lifecycle: Majors/Programs (pure data), Trainees, sticky Academy Mode sessions, resource proposals, the CENTRAL deduplicated shared specialist corpus (sources/specialists/provenance/subscriptions), promotion, capsule refresh, Trainer review (deterministic + live LLM-router client), curation staging, `stage_academy_apply` (fail-closed apply contract), continuing-education plan builder, adopt/browse/search surfaces.
- `python/arclink_academy_scheduler.py` (1067 lines) — weekly forward-maintenance CLI/job. **Performs real bounded public-HTTPS live crawling** (SSRF-guarded, robots-respecting) to observe source freshness, records crawl-observation/event/audit/notification rows, refreshes central capsules, optionally runs the live Trainer review, but writes NO Agent files.
- `python/arclink_academy_trainer.py` (2380 lines) — no-network/no-write schemas + fail-closed planning helpers: 8-lane registry, dataclasses, `validate_academy_sources`, `build_academy_corpus`, scoring, evaluation/graduation gates, `build_agent_application_plan`, `build_continuing_education_plan`, `build_academy_review_status`, and the application-preview worker-boundary validators.
- `python/arclink_crew_recipes.py` (1508 lines) — Crew presets/capacities, deterministic recipe overlay, injectable proof-gated provider generation, `apply_crew_recipe` (archive+insert+per-deployment identity projection), and the Crew-Recipe `academy_training` overlay (single-review + per-Agent rollup shapes).
- `templates/CREW_RECIPE.md.tmpl` (20 lines) — the provider prompt template (`string.Template` `$var` substitution).
- `templates/SOUL.md.tmpl` (110 lines) — the tenant Agent SOUL identity overlay (consumed by onboarding/provisioning, NOT by this piece's code).
- `templates/SOUL.operator.md.tmpl` (15 lines) — the Operator Hermes SOUL stance.

All four listed `tests/test_arclink_academy_*.py` + `tests/test_arclink_crew_recipes.py` exist (64 tests, all pass). The Academy schema lives in CANON-01 (`python/arclink_control.py:1476-1709`): **10 `academy_*` tables** + `arclink_crew_recipes` (the prior ground-truth doc undercounted — it missed `academy_source_crawl_observations`).

**UNBUILT-vs-real verdict:** The prior audit's "scaffold-only / UNBUILT core vision" is now STALE. The central shared-SME corpus, live LLM-router Trainer client, live weekly crawler, and the real SOUL/vault/qmd/skill materialization seam are all CODE-PRESENT and test-covered. What remains externally-gated is only authorization (`PG-PROVIDER`/`PG-HERMES` env flags) and source-governance policy — not absent code.

## ADVERSARIAL SELF-CHECK
1. **"The live crawler actually runs in production."** Verified the code path and the compose default (`ARCLINK_ACADEMY_CE_LIVE_CRAWL` default True, weekly interval). Falsifier: an operator override or a `*.env` setting `ARCLINK_ACADEMY_CE_LIVE_CRAWL=0`, or the CE compose service being disabled, would mean it never fetches. I did not inspect `config/*.env*` for an explicit override.
2. **"`stage_academy_apply` stale-contract gate cannot be bypassed."** I traced the staged-vs-recomputed id comparison (`contract_fresh`, `programs.py:2851`) and the `writes_enabled` decision. Falsifier: if `_compose_trainee_corpus` is non-deterministic across runs (e.g. fixture fallback ordering, or `read_central_specialist_sources` ordering), a graduated trainee could spuriously go `stale_requires_regraduation`. The SQL orders by `first_seen_at, source_uid` and uses a stable timestamp, so I believe it's deterministic, but I did not run a two-graduation differential test.
3. **"Both ends of the action-worker SOUL seam match."** I read both producer and consumer key sets. Falsifier: the consumer also reads `program_id`/`academy_specialist_uid` for slug/state — if `stage_academy_apply` ever omitted `program_id` the vault base would silently fall back; it always emits it (`programs.py:2961`), so low risk.
4. **"Central promotion never leaks `organization_private`."** `share_eligible_source_lanes` excludes `organization_private` (`trainer.py:649`), and promotion skips non-eligible lanes (`programs.py:1741`). Falsifier: a Major whose only lanes are private would create no central specialist — verified the `has_candidate` guard (`programs.py:1657`). I'm fairly sure here.
5. **"The `Sequence` import bug has no runtime impact."** Verified via live `get_type_hints` failure + successful `_academy_rollup_status` call. Falsifier: any future caller using `pydantic`/`dataclasses`-style hint resolution on that function would crash; none found in this piece.
