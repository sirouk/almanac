# CANON-23 — Diagnostics / Health / Evidence / Notifications / Live Proof

## PIECE
This piece is ArcLink's secret-safe observability and proof spine. It owns exactly seven tracked modules:
`python/arclink_secrets_regex.py` (122 lines — shared secret-detection/redaction regex helpers),
`python/arclink_diagnostics.py` (230 — presence-only provider credential diagnostics + qmd embed-staleness alert),
`python/arclink_health_watch.py` (310 — runs `bin/health.sh`, edge-triggered operator notifications on fingerprint change),
`python/arclink_evidence.py` (376 — secret-redacted evidence ledger model + `arclink_evidence_runs` DAL),
`python/arclink_live_journey.py` (416 — ordered `JourneyStep` model + four journey catalogs and an in-place evaluator),
`python/arclink_live_runner.py` (829 — six-phase live-proof orchestrator composing readiness+diagnostics+journey+evidence, plus real no-secret workspace/router runners; CLI behind `bin/arclink-live-proof`),
`python/arclink_notification_delivery.py` (1966 — the `notification_outbox` delivery worker, six target_kinds, the detached Hermes-gateway public-agent bridge).
Together they: probe credential/host readiness without printing secret values; plan and (for `workspace`/`router` journeys) execute live proof runs writing redacted JSON to `evidence/<run_id>.json`; watch health and alert the operator; and drain the outbox to Discord/Telegram/TUI/agent bridges.

## RISKS (severity-ranked, code-cited)
- **MEDIUM — No retry backoff on delivery failures.** `mark_notification_error` leaves `next_attempt_at` untouched (`arclink_control.py:9403-9408`); with the delivery timer at `OnUnitActiveSec=5s`, a persistently failing non-leased row (e.g. operator telegram with a bad token) is re-attempted every 5s indefinitely, hammering the external API. Only `public-agent-turn` rows get leased/back-pressured (`:1629-1649`).
- **MEDIUM — Evidence ledger DB layer is unwired (governance gap).** `store_evidence_run`/`latest_evidence_status` are fully implemented + table exists (`arclink_evidence.py:278-376`, `arclink_control.py:2523`) but `run_live_proof` only writes a local JSON file (`arclink_live_runner.py:743-748`) and no surface reads the table. The symphony's "operator-visible evidence state" is not real.
- **MEDIUM — Two divergent redaction engines.** `arclink_evidence._SECRET_PATTERNS` (`:26-31`) covers far fewer secret families (no PEM/JWT/Anthropic `sk-ant`/Chutes `cpk_`/Telegram-token) than `arclink_secrets_regex.PLAINTEXT_SECRET_RE` (`:25-44`); evidence does not import the shared module. A secret family present in a journey `detail` but absent from `_SECRET_PATTERNS` could survive into `evidence/<run_id>.json` unless it also hits `_REDACT_ENV_KEYS`/`_is_sensitive_key`.
- **MEDIUM — Live-runner Phase 4 mutates global `os.environ`.** `run_live_proof` does `os.environ.update(source)` then restores in `finally` (`:705-711`) because `evaluate_journey`/`missing_credentials` read `os.environ` not the passed `env`. Not thread-safe; a concurrent thread sees injected creds during a live run.
- **LOW — Substring container allowlist.** `_validate_public_agent_bridge_cmd` requires `"hermes-gateway" in container_name` (`:495`) rather than an exact service-name match; an attacker-influenced compose project named to embed that substring could satisfy it, though `PUBLIC_AGENT_BRIDGE_CONTAINER_RE` and project-prefix checks narrow this (`:492-499`).
- **LOW — `diagnose_qmd_pending_embeddings` swallows clock skew.** Uses wall-clock `time.time()` minus a marker epoch (`:175-177`); a marker written with a future epoch yields `age=0` (clamped, `:176`) and silently reports healthy.
- **INFO — `run_diagnostics(live=...)` is a no-op param** (`:200-213`); callers passing `live=True` get identical output, which could mislead operators into thinking connectivity was tested.

## VERDICT
This piece provably does its core job: dry-run live proof, presence-only diagnostics, edge-triggered health alerting, and outbox delivery are all real, code-verified, and secret-safe on the happy path. Load-bearing strengths: (1) credential diagnostics emit names not values with no value-leak path; (2) evidence serialization re-redacts on every `to_dict`; (3) bridge secrets travel via stdin/header, never argv, behind an exact-shape allowlist + symlink/path-within preflight; (4) health-watch fingerprint de-duplication and deploy-window suppression are correctly implemented; (5) the workspace and router live runners are genuinely executable with no secrets and verified seams into the dashboard/router. Real weaknesses: the evidence-ledger DB persistence is built but never wired (the largest delta vs the "operator-visible evidence" vision); delivery failures have no backoff and can hot-loop external APIs every 5s; redaction is split across two engines with unequal coverage; and the runner mutates process-global `os.environ` mid-run. The `external` journey remains a catalog (no runners), and `run_diagnostics(live=)` is a dead parameter — both must not be read as "live provider proof exists."
