# CANON-26 — Systemd Services & Timers

## PIECE
This piece owns the **systemd unit semantics** for ArcLink: the 29 static unit files under `systemd/user/` (18 `.service` + 11 `.timer`) plus the 5 installer scripts that render/enable units (`bin/install-user-services.sh`, `bin/install-system-services.sh`, `bin/install-agent-user-services.sh`, `bin/install-agent-cron-jobs.sh`, `bin/install-agent-ssh-key.sh`). The static units are **all user-level** (no `systemd/system/` directory exists — `ls systemd/` shows only `user/`). Each static unit's `ExecStart` is `%h/arclink/bin/<script>` (verified: `grep ExecStart systemd/user/ | grep -v %h` returns nothing), so every unit is a thin systemd wrapper around a `bin/` script owned by an adjacent piece (CANON-31/CANON-22/CANON-25/CANON-06). This piece defines: **what each unit runs**, **timer cadence**, **dependency ordering**, **Type/Restart semantics**, and **the install-time enable/disable gating**. `install-system-services.sh` additionally generates 4 *system*-level units inline (`/etc/systemd/system`) at install time — these are not tracked files but their unit semantics are owned here. **Boundary with CANON-24:** CANON-24 owns the deploy/install ORCHESTRATION that *invokes* these installers; CANON-26 owns the UNIT SEMANTICS (ExecStart targets, Type, Restart, timer cadence, ordering) and the installer logic itself.

## RISKS (severity-ranked, code-cited)
- **MEDIUM** — `install-agent-user-services.sh:287` reads `state["dashboard_backend_port"]` as a hard dict subscript; if the access-state JSON is malformed or schema-renamed, the whole per-agent install aborts with `KeyError` (no `.get`/default). Fail-closed but brittle across the CANON-19 boundary. Cite `install-agent-user-services.sh:286-289`.
- **LOW** — Per-agent dashboard ports are written by multiple producers (`arclink_agent_access.py`, `arclink_agent_process_helper.py`, `arclink_docker_agent_supervisor.py`, `arclink_control.py` all reference `dashboard_backend_port`/`dashboard_proxy_port`); the install-time hard subscript (`install-agent-user-services.sh:287`) depends on whichever wrote `arclink-web-access.json`. Key drift in any one producer would KeyError the install. Cite `install-agent-user-services.sh:286-289`.
- **LOW** — `start_system_service_if_idle` (`install-system-services.sh:110-119`) checks `ActiveState` then `start` non-atomically; a concurrent install could double-start. Low impact (oneshot idempotent-ish) but a real TOCTOU. Cite `:111-118`.
- **LOW** — `install-user-services.sh` mutates live systemd state with `systemctl --user enable/restart` and `set -e`; a single failing unit restart (`:109-120` lacks per-unit `|| true`) aborts the whole installer, potentially leaving units half-enabled. Cite `install-user-services.sh:36-40,109`.
- **INFO** — `arclink-pdf-ingest-watch.service` and `arclink-user-agent-code.service` are shipped/referenced but always disabled/removed (`install-user-services.sh:72`, `install-agent-user-services.sh:442`); dead units that can confuse operators. Cite those lines.
- **INFO** — `install-system-services.sh` injects `Environment=HOME=/root` unconditionally (`:58,85`); correct only when the service user is root, which it requires (`:8`), but hardcodes `/root` rather than deriving it. Cite `:58`.
- **INFO** (strength) — Both inline renderers guard against unit injection: `reject_systemd_unit_value` rejects newline/CR/`$` (`install-system-services.sh:18-30`, `install-agent-user-services.sh:27-37`) and `systemd_quote_value` escapes `\ " %` (`:32-38`/`:39-45`).
