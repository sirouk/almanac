# CANON-29 — Test Corpus

## PIECE
This piece owns the `tests/**` tree: **128** tracked `tests/test_*.py` regression files, one shared helper module `tests/arclink_test_helpers.py`, and one golden fixture `tests/fixtures/arclink_traefik_labels.golden.json`. (The CANON-29 brief said "129 tests"; the real tracked count is 128 — `git ls-files 'tests/test_*.py' | wc -l` = 128. Off-by-one likely counts the helper module or a web test.) The corpus is a **no-secret, in-memory, runnable-as-`__main__`** regression suite: each `tests/test_*.py` is invoked directly by CI as `python3 tests/test_x.py` (`.github/workflows/install-smoke.yml:38-40`), NOT via `pytest` collection despite `pytest.ini` existing. The corpus characterizes the whole Python control plane: it loads the real `python/arclink_*.py` modules by file path (via `tests/arclink_test_helpers.py:19-29` `load_module` or per-file copies of the same helper) and exercises them against `sqlite3.connect(":memory:")` databases seeded by `control.ensure_schema()` (`tests/arclink_test_helpers.py:32-36`). The fake-vs-live split is enforced by two harnesses: `tests/test_arclink_e2e_fake.py` (fakes + in-memory SQLite, no creds) and `tests/test_arclink_e2e_live.py` (gated on `ARCLINK_E2E_LIVE=1` + per-provider credential env vars, skips cleanly otherwise). **In scope but boundary-owned elsewhere:** the web tests under `web/tests/**` (`test_api_client.mjs`, `test_page_smoke.mjs`, `browser/product-checks.spec.ts`) physically live outside `tests/` and are run by the `web-regressions` CI job (`npm test`/`npx playwright test`); their source files are CANON-03 (web/src) / CANON-28 (CI), but I characterize their seam here because the brief names "web tests."

## INPUT CONTRACT (code-verified)

- **`tests/arclink_test_helpers.py` API** (imported as top-level module because CWD/`sys.path[0]` is repo root or the file's dir):

## OUTPUT CONTRACT (code-verified)
- **Process exit code** is the contract CI consumes. `unittest.main()` exits non-zero on failure; `raise SystemExit(main())` exits with `main()`'s int (0 on success); hand-list runners propagate the first raised `AssertionError` as a non-zero exit. CI's `set -euo pipefail` (`.github/workflows/install-smoke.yml:31`) fails the job on the first non-zero `python3 "$test_file"`.
- **stdout PASS markers**: non-unittest tests print `PASS <fn name>` per success (e.g. `tests/test_arclink_e2e_fake.py:364`) and a summary line (e.g. `"All fake E2E tests passed."` :377, `"All 24 fleet tests passed."`).
- **Side effects**: tests write only to `:memory:` SQLite (`memory_db`) or `tempfile`/`TemporaryDirectory` scratch (e.g. `test_arclink_operator_raven.py:8`, `test_arclink_surface_contract.py:11` import `tempfile`). No mutation of the real repo, no network in CI mode. `test_arclink_e2e_live.py` makes read-only HTTP/Docker calls ONLY when gated (catalog GET, `getMe`, `docker compose version` :221-231) and asserts no destructive mutation.
- **Golden-file output**: `tests/fixtures/arclink_traefik_labels.golden.json` is the asserted-equal expected value for `arclink_ingress.render_traefik_dynamic_labels(...)` (consumer `tests/test_arclink_ingress.py:72-74`).
- **Meta-test outputs** police the repo: `test_public_repo_hygiene.py` (scans tracked+untracked text for private operator names/secrets, `:158`,`:176`) and `test_documentation_truths.py` (validates `research/PRODUCT_REALITY_MATRIX.md` row totals/statuses, `:76-88`).

## CODE vs COMMENT/DOC/NAME DRIFT

Tests that rely on pytest collection (any `test_*` not wired into the file's `__main__`/`main()`) are silently NOT run by CI. CODE WINS: the suite's real contract is "runnable `__main__`", not "pytest-collectable".
