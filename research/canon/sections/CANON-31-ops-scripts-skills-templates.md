# CANON-31 — Operational & Knowledge-Pipeline Scripts, Skills & Templates

## PIECE
This piece is the catch-all for ArcLink's runtime OPS shell/python scripts plus the agent-facing `skills/` and `templates/` trees that are NOT claimed by a Python-owning sibling piece. It owns three classes of artifact:

1. **Thin exec wrappers** under `bin/` that `source bin/common.sh`, resolve a runtime python/hermes, set `PYTHONPATH`, and `exec` into a Python module owned by an adjacent piece. These are the systemd `ExecStart=` entrypoints. Examples: `bin/memory-synth.sh` -> `python/arclink_memory_synthesizer.py` (CANON-18), `bin/arclink-ssot-batcher.sh` -> `python/arclink_ssot_batcher.py` (CANON-18), `bin/arclink-mcp-server.sh` -> `python/arclink_mcp_server.py` (CANON-18), `bin/arclink-notion-webhook.sh` -> `python/arclink_notion_webhook.py` (CANON-18), `bin/arclink-notification-delivery.sh` -> `python/arclink_notification_delivery.py` (CANON-23), `bin/arclink-wrapped.sh` -> `python/arclink_wrapped.py` (CANON-22), `bin/health-watch.sh` -> `python/arclink_health_watch.py` (CANON-23), `bin/org-profile-builder.sh` -> `python/arclink_org_profile_builder.py` (CANON-21), `bin/arclink-enrollment-provision.sh` -> `python/arclink_enrollment_provisioner.py` (CANON-08), `bin/vault-repo-sync.sh` / `bin/curator-refresh.sh` -> `python/arclink_ctl.py internal …` (CANON-14), `bin/hermes-shell.sh`/`bin/hermes-upgrade.sh` (the latter -> `bin/component-upgrade.sh`, CANON-24).

## CODE-PATH TRACE (end-to-end: a PDF lands in the vault)
1. Operator/agent drops `Foo.pdf` under `$VAULT_DIR`. `inotifywait` (started at vault-watch.sh:340-348) emits `…/Foo.pdf|CREATE` to the watcher's fd.

3. `vault_watch_need_pdf` → `PDF_INGEST_TRIGGER_QMD_REFRESH=0 pdf-ingest.sh --quiet` (vault-watch.sh:433).
4. `pdf-ingest.sh` checks `PDF_INGEST_ENABLED==1` (pdf-ingest.sh:29), exports the PDF_* env, acquires `flock $PDF_INGEST_LOCK_FILE`, and runs `pdf-ingest.py` capturing its STDOUT JSON (:37-60).
5. `pdf-ingest.py main()` resolves backend (`docling`|`pdftotext`, :64-77,:599), lists vault PDFs with a symlink+commonpath traversal guard (`list_pdf_sources`, :506-527), and for the unchanged-detection compares `source_sha256` + `pipeline_signature` + `status='ok'` + artifact existence (:656-666). New/changed → `extract_text_body` + optional `generate_visual_notes` (vision), `render_markdown` writes `Foo-pdf.md` (:711-726) inside the `require_generated_artifact_path` root guard (:147-151), and upserts the manifest row (:731-769). STDOUT = summary JSON; STATUS_FILE mirrors it (:835-836).
6. Back in vault-watch.sh, `pdf_status_needs_qmd_refresh` reads `qmd_refresh_needed` from STATUS_FILE (vault-watch.sh:111-123) → sets `vault_watch_need_qmd=1` (:434-436).

8. `qmd-refresh.sh` takes `flock 9`, runs `configure_qmd_collections` (which `ensure_qmd_collection PDF_INGEST_COLLECTION_NAME $PDF_INGEST_MARKDOWN_DIR '**/*.md'` when PDF ingest is on, common.sh:1149-1153), `qmd … update`, optional embed (qmd-refresh.sh:122-135). ## RISKS (severity-ranked, code-cited)
- **MEDIUM — Misleadingly-named teardown script.** `bin/tailscale-nextcloud-serve.sh` no longer serves; it un-serves (tailscale-nextcloud-serve.sh:235-240). An operator (or future code) invoking it to publish Nextcloud/MCP gets silent teardown instead. ~200 lines of live-looking serve helpers are dead code, inviting confusion.
- **MEDIUM — Endpoint embeddings silently disabled.** `QMD_EMBED_PROVIDER=endpoint` (and aliases) falls back to local embeddings with only a stderr warning (qmd-refresh.sh:71-79). On a host configured for remote embeddings, vector quality/cost expectations silently differ from config; the warning is easy to miss in a timer log.
- **MEDIUM — vault-watch fail-open on unreadable manifest.** `directory_tree_had_manifest_pdf` returns success (treat-as-had-pdf) when the manifest DB exists but can't be read (bin/vault-watch.sh:78-81, `raise SystemExit(0)`). Comment says this is deliberate (let the reconciler clean up), but it means a corrupt manifest triggers unnecessary PDF reconciliation churn on every directory delete.
- **LOW — Momentary bootstrap-token argv exposure.** Skill helpers pass the token as python heredoc argv (curate-vaults.sh:67-72, curate-notion.sh:90-100, run-first-contact.sh:42-46). Visible in `ps` for the lifetime of the (sub-second) python process on a shared host. Low because loopback + ephemeral, but it is a real TOCTOU-adjacent exposure.
- **LOW — qmd embed timeout is swallowed as success.** A 124/137 timeout returns 0 ("text index updated, embeddings retry next refresh", qmd-refresh.sh:105-108). A permanently-starved embed lane only surfaces via the separate `qmd_pending_embeddings_age_alert` health marker (common.sh:1096-1133), not by failing the refresh job — vector search can stay stale for ≥6h before any warning.
- **LOW — `pdf-ingest.py` requires env or hard-crashes.** `os.environ["VAULT_DIR"]` etc. raise KeyError (uncaught) if the wrapper didn't export them (bin/pdf-ingest.py:21-24). Fine in the wrapped path, but the script is unusable standalone without the four env vars — no friendly error.
- **INFO — `notion-page-pdf-export.py` depends on un-vendored playwright/requests** (notion-page-pdf-export.py:49-50) and is intentionally NOT in any default skill install list; it is operator-enabled, chromium-gated. No risk in default deployment; just a hidden dependency.
- **INFO — `clone-team-resources.sh` does `git reset --hard` on existing checkouts** (clone-team-resources.sh:16-17 doc) under the operator manifest. Destructive to local edits by design; acceptable for a managed resource library but worth noting it mirrors the same hard-sync hazard documented in `templates/.../Repos/README.md`.

## VERDICT
This piece **provably does its job**: it is the glue layer that turns ArcLink's Python control plane into a running knowledge/ops pipeline. Load-bearing strengths: (1) the qmd lifecycle (`qmd-daemon`/`qmd-refresh`/`vault-watch`) is correctly lock-serialized (`flock 9`) and self-healing (event-stream restart, pending-embed marker), with a verified end-to-end PDF→sidecar→qmd→synth→retrieval chain; (2) `pdf-ingest.py` is a genuinely robust standalone pipeline with content-hash freshness (not size+mtime), a traversal-guarded artifact root, a secret-minimized subprocess env, and an idempotent SQLite manifest with stale-row reconciliation; (3) every thin wrapper's exec seam and every skill→MCP tool seam was verified at both ends against the real `arclink_mcp_server` TOOLS table and `arclink_ctl internal` subparsers; (4) the skills consistently and correctly teach token-omitted brokered tool use and credentials-omitted resource refs, matching the code. Real weaknesses: the **named-but-inverted** `tailscale-nextcloud-serve.sh` (now a teardown), the **silently-downgraded** endpoint-embedding provider, several **fail-open / swallow-timeout** paths in vault-watch and qmd-refresh that trade strict correctness for liveness (defensible but under-alarmed), and a momentary token-on-argv exposure in the agent skill helpers. Two seams remain not-both-ends-verified by design: the external `hermes_cli.config` consumer and the `vault notify-paths`/`reload-defs` CLI handlers (producer verified, consumer assumed-present).
