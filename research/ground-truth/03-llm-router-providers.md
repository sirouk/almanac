## (a) What is actually implemented today (local-real)

### Router service shape
- `arclink_llm_router.py` builds a **FastAPI** app via `create_app()` and exposes a module-level `app = create_app()`. Run as `uvicorn arclink_llm_router:app` on port **8090** (see compose).
- Routes:
  - `GET /health` — no auth; returns `RouterConfig.public_status()` plus `model_catalog_refresh` state. 200 when configured, 503 otherwise.
  - `GET /v1/models` — Bearer router key; returns the key's `allowed_models` (else `config.allowed_models`) as OpenAI-style model list, `owned_by: "arclink-chutes"`.
  - `POST /v1/chat/completions` — Bearer router key; full policy + reservation + relay + settlement.
- On startup it runs `_refresh_model_catalog_once` in a thread (when `refresh_model_catalog_on_startup`), storing result in `app.state.router_catalog_refresh`.

### Config (`RouterConfig` / `load_router_config`)
Frozen dataclass loaded from env (precedence `ARCLINK_*`). Defaults that are real in code:
- `DEFAULT_CHUTES_BASE_URL = "https://llm.chutes.ai/v1"`, `DEFAULT_MODEL = "moonshotai/Kimi-K2.6-TEE"`.
- `DEFAULT_INPUT_CENTS_PER_MILLION = 95`, `DEFAULT_OUTPUT_CENTS_PER_MILLION = 400`.
- `DEFAULT_MAX_BODY_BYTES = 1MiB`, `DEFAULT_PROMPT_ESTIMATE_TOKEN_CAP = 120000`, `DEFAULT_MAX_TOKENS_CAP = 8192`, `DEFAULT_DEPLOYMENT_CONCURRENCY_LIMIT = 4`.
- Rate limits: key 60/min, deployment 120/min, user 300/min.
- `DEFAULT_MIN_RESERVATION_CENTS = 1`.
- Fallback status codes default `(429, 500, 502, 503, 504)`.
- `configured` is `True` when disabled, else requires non-blank `chutes_api_key` AND `db_path`.

### Authentication / per-deployment router keys
- `_authenticate_request` extracts a `Bearer` token, opens the control SQLite DB, calls `verify_llm_router_key(conn, raw_key)` (in `arclink_control.py`).
- Key format (`arclink_control.py`): prefix `acpod_live_`, regex `^acpod_live_([A-Za-z0-9]{8,32})_([A-Za-z0-9_-]{32,})$`; key id derives as `llmk_<group1>`. Raw key generated by `generate_llm_router_raw_key()` = `acpod_live_<hex12>_<token_urlsafe36>`.
- Hashing: **HMAC-SHA256** with pepper `ARCLINK_LLM_ROUTER_KEY_HASH_PEPPER` → falls back to `ARCLINK_SESSION_HASH_PEPPER` → dev default. Stored hash prefixed `hmac-sha256$`. Legacy plain `hash_token` (SHA-256) rows accepted ONLY for verification migration (`_legacy_hash_llm_router_key`); table query matches `key_hash IN (current, legacy)`.
- `verify_llm_router_key` joins `arclink_deployments` + `arclink_users`; rejects when row count != 1, status != active, deployment status in `{cancelled, torn_down, teardown_complete}`, or user status != active. Returns sanitized record (key_hash stripped, `allowed_models` + `metadata` decoded). It also rewrites legacy rows to the HMAC format on successful verify (touches `last_seen_at`).

### Request policy / preflight (`_preflight_chat_request`)
In order:
1. Require `model` in body (400 `missing_model`).
2. Allowlist check `_router_model_allowed` against key's `allowed_models` (else `config.allowed_models`); also permits `config.default_model` even if it's a CSV string (provider-side fallback). 403 `model_not_allowed`.
3. Resolve model `_resolve_router_model` (replacements / catalog deprecation / auto-promote — see below).
4. Prompt token estimate `_estimate_prompt_tokens` ≈ `len(body)/4`; cap → 413 `prompt_too_large`.
5. `max_tokens`/`max_completion_tokens` cap → 400 `max_tokens_too_large`.
6. Load deployment context + billing state; evaluate `evaluate_chutes_deployment_boundary` (from `arclink_chutes.py`). If `not allow_inference`: 402 for `{billing_suspended, budget_unconfigured, budget_exhausted}`, else 403, with `credential_state` as error code. On `budget_exhausted`, queues a fuel notice (severity `empty`).
7. Rate limits per `llm-router:key` / `:deployment` / `:user` via `rate_limit_count` (60s window) → 429 `rate_limited` Retry-After 60.
8. Concurrency cap: `_open_reserved_count` (status `reserved` rows for deployment) ≥ `deployment_concurrency_limit` → 429 `concurrency_limited`.
9. Fallback-aware reservation pricing `_fallback_reservation_pricing`: prices EVERY fallback candidate, selects the **max reserved_cents** so the reservation covers the most expensive candidate. If `boundary.remaining_cents < reserved_cents` → 402 `budget_exhausted` + low-fuel notice.
10. Record rate-limit events, create reservation `_create_budget_reservation` (status `reserved`, id `llmres_...`, request id `llmreq_...`).

### Reservation vs settlement pricing (real)
- Reservation = max over candidates of `_estimate_reservation_cents_for_model` (input_tokens*input_cents + output_tokens*output_cents, ceil-divided by 1e6, floored at `min_reservation_cents`). Output token estimate = requested max_tokens or `min(max_tokens_cap, 1024)`.
- Pricing source per model = catalog entry `input_cents_per_million`/`output_cents_per_million` when both > 0, else env/router default (`_model_price_cents` → `"catalog"` or `"router_default"`).
- Settlement (`_record_router_usage`): re-prices using the **final model actually used** (`get_model_catalog_entry(model=final_model)`); `actual_cents` only computed for `status == "succeeded"`. Reservation is released as `settled` (succeeded) or `failed`. `pricing_adjusted_at_settlement = (final_model != reservation_pricing_model)`.

### Non-streaming relay (`_forward_non_streaming`)
- Iterates `_router_fallback_candidates` (primary + `config.fallback_models`, deduped). Uses `httpx.AsyncClient` (transport injectable via `app.state.router_upstream_transport`; 60s timeout). POSTs `<base>/chat/completions` with `_prepare_upstream_payload` (forces `stream_options.include_usage=True` when streaming) and `_upstream_headers` (`Authorization: Bearer <central chutes key>`).
- On `httpx.HTTPError`, status ≥ 400 (if `_upstream_status_is_retryable`), or invalid JSON: records a sanitized `_fallback_attempt_metadata`, emits a `llm_router:fallback_attempt` event, advances to `next_model` if any; else settles `failed` and returns router error (`upstream_unavailable` 502 / `upstream_error` / `invalid_upstream_response` 502).
- On success: parses usage `_usage_from_payload`, injects `arclink_router` metadata block when fallback used, settles `succeeded`.

### Streaming relay (`_stream_upstream_response`)
- SSE `StreamingResponse` (`text/event-stream`). Pre-stream retry: if upstream status ≥ 400 before any chunk and retryable → `streaming_fallback="pre_stream"`, advance candidate. When a fallback candidate is chosen, emits a leading `arclink_router` SSE metadata frame.
- After first chunk yielded (`yielded_any`), NO replay: on later error it labels `streaming_fallback = "unavailable_after_stream_started"` / outcome `failed_after_stream_started`, emits a sanitized error SSE frame, settles. Usage scraped from SSE chunks via `_usage_from_sse_chunk` (looks for `usage` in `data:` frames). Settlement always runs in `finally`.

### Usage recording (sanitized)
- `arclink_llm_usage_events` row written per request (succeeded or failed): request_id, deployment_id, user_id, provider `chutes`, model, token counts, estimated/actual cents, status, stream, source_kind, **redacted** error_summary (`redact_then_truncate` via `arclink_secrets_regex`), and a metadata_json block with requested/primary/usage/final models, fallback flags, fallback_attempts (public), pricing models+sources, reserved/settled cents. **No raw prompts or completions stored.**
- On success it also calls `record_chutes_usage_event` (in `arclink_chutes.py`) which advances the deployment metadata `chutes.used_cents` ledger (idempotent by event id hash) and may queue a fuel notice.

### Fuel notices (refuel nudges) — real and local
- `_queue_arc_pod_fuel_notice` writes to `notification_outbox` (`target_kind='public-bot-user'`, telegram/discord) with a **Refuel ArcPod** button (`callback_data`/`custom_id` = `arclink:/raven refuel`), deduped via `llm_router:arc_pod_fuel_notice_queued` events keyed by `severity:budget:warning_threshold`. Resolves the Captain's public channel from `arclink_onboarding_sessions`. Wrapped in try/except so notices never block inference.

### Model catalog + promotion (real)
- `_refresh_model_catalog_once` uses `ChutesCatalogClient.list_models` (auth strategies `bearer`/`x-api-key`/`none`) → `upsert_model_catalog(provider="chutes", mark_missing_unavailable=...)`.
- `arclink_model_catalog` table (PK provider+model_id) stores capabilities, confidential_compute, input/output cents, status CHECK(`active|deprecated|unavailable`), replacement_model_id, family, version_sort_key, first/last_seen_at, raw_json.
- `_resolve_router_model` precedence: env `model_replacements` → catalog `replacement_model_id` for deprecated/unavailable → auto-promote to `latest_model_in_family` when newer version_sort_key or non-active. Records `replacement_reason` (`env_replacement` / `catalog_*` / `latest_family_model` / `newer_family_model`).

### Chutes deployment boundary (`arclink_chutes.py`, real, local)
- `evaluate_chutes_deployment_boundary` returns `ChutesDeploymentBoundary`. Accepted isolation modes: `per_deployment_secret_ref`, `per_user_secret_ref`, `per_user_chutes_account_oauth`. Rejects: operator shared key (`operator_shared_key_rejected`), plaintext (`plaintext_secret_ref_rejected`), unscoped `secret://` (`unscoped_secret_ref_rejected`). Requires scoped `secret://...` containing deployment_id or user_id PLUS a configured budget. Fail-closed states: `missing_secret_ref`, `invalid_secret_ref`, `unscoped_secret_ref`, `budget_unconfigured`, `budget_exhausted`, `billing_suspended`, `suspended`, `account_oauth_required`.
- Budget math: warning default 80%, hard limit default 100%; `budget_status` ∈ `unconfigured|ok|warning|exhausted`.
- `record_chutes_usage_event` reduces events to cents+tokens+safe ids before storage; idempotent via sha256 identity; never persists raw payloads/headers/secret refs.

### Compose wiring — `control-llm-router` IS wired
`compose.yaml` defines service `control-llm-router` (lines 511–537): `uvicorn arclink_llm_router:app` on port 8090, healthcheck `GET /health`, full env block. `control-ingress` `depends_on` it with `condition: service_healthy`. ArcPod default `ARCLINK_CHUTES_BASE_URL` points at `${ARCLINK_LLM_ROUTER_PUBLIC_BASE_URL:-http://control-llm-router:8090/v1}` (compose line 191). **Note:** compose sets `ARCLINK_LLM_ROUTER_DEFAULT_MONTHLY_BUDGET_CENTS` default to **2500**, whereas the router code default is **0**.

## (b) Proof-gated / fake-adapter / local-only

- **Live Chutes inference is never called in tests/CI.** All router tests inject `app.state.router_upstream_transport` (fake async transport). Live proof gated by `ARCLINK_LLM_ROUTER_LIVE_CHUTES_PROOF=1` + `PG-PROVIDER` (GAP-031).
- **`arclink_chutes_live.py` (`ChutesLiveAdapter`) is TEST-ONLY / NOT WIRED.** Repo-wide search shows it is imported only by `tests/test_arclink_chutes_live_adapter.py` and `tests/test_public_repo_hygiene.py`. No production module (router, hosted API, provisioning, entitlements) imports it. It is a secret-ref boundary over Chutes `/models`, `/users/me`, `/users/me/subscription_usage`, `/users/{id}/usage`, quotas, discounts, price_overrides, `/api_keys/`, `/idp/scopes`, `/idp/token/introspect`, `/users/balance_transfer`. Mutations (`create_api_key`, `delete_api_key`, `transfer_balance`) gated by `allow_live_mutation` else raise "proof-gated". Ships `FakeChutesLiveTransport` with fixtures.
- **`arclink_chutes_oauth.py` is TEST-ONLY / NOT WIRED.** Imported only by `tests/test_arclink_chutes_oauth.py` and `tests/test_public_repo_hygiene.py`. PKCE S256 connect/callback with `InMemory*` state/token stores and `FakeChutesOAuthCodeExchanger`. `ChutesOAuthConnectPlan.to_public()` returns `connect_status: "proof_gated_until_authorized_oauth_client_is_configured"`. `disconnect_chutes_oauth(revoke_live=True)` raises proof-gated. No live OAuth client configured, no callback route in the router or hosted API.
- **`FakeChutesKeyManager` / `FakeChutesInferenceClient`** (in `arclink_chutes.py`) are fake smoke-test helpers; the router itself does NOT use them — it relays directly to the upstream base URL with the central key.
- **Chutes official account registration** (`build_chutes_official_registration_request`) is modeled secret-safely; `to_public()` reports `live_status: "proof_gated_until_operator_supplies_authorized_token_and_hotkey"`. `reject_chutes_browser_challenge_bypass` explicitly refuses Cloudflare/hCaptcha bypass.
- **`per_user_chutes_account_oauth`** isolation mode is recognized by the boundary as a fallback lane "required when per-key metering unavailable" — but there is NO code path that actually performs OAuth-backed inference; it's a posture/label only.
